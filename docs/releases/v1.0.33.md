# Entroly v1.0.33

Status: release-ready after 15 successful checks and 5 skipped publish/release jobs.

## Highlights

Entroly v1.0.33 introduces the first stable release cut for Evidence-Locked Compression and the compression proxy work.

This release focuses on making Entroly a high-efficiency, auditable, recoverable context-compression control plane for agentic AI systems.

## New: Evidence-Locked Compression

Evidence-Locked Compression compresses aggressively around evidence, never through evidence.

It preserves:

- error, warning, failure, traceback, and timeout anchors,
- nearby context around important lines,
- file paths and line references,
- query-specific entities,
- JSON schema, query matches, and outliers,
- omitted-span receipts for recoverability.

## New: Compression proxy surface

The new compression proxy API supports provider-light payload compression for OpenAI-style and Anthropic-style message payloads.

Public APIs:

```python
from entroly import compress_proxy_payload
from entroly import compress_proxy_payload_from_env
```

Proxy metadata headers are available through `ProxyCompressionResult.headers()`:

```text
x-entroly-compression-mode
x-entroly-original-tokens
x-entroly-compressed-tokens
x-entroly-tokens-saved
x-entroly-savings-ratio
x-entroly-compressed-blocks
```

## New: recoverable omitted-span store

Entroly now includes a local retrieval store for compressed omitted spans.

Public APIs:

```python
from entroly import CompressionRetrievalStore
```

The store supports:

- storing omitted spans from ELC receipts,
- retrieving by receipt id and span id,
- searching omitted spans by keyword,
- persisting locally to disk.

## New: compression retrieval MCP server

Entroly now ships a focused MCP server for retrieving compressed spans:

```bash
entroly-compression-mcp
```

Tools exposed:

```text
retrieve_compressed_span(receipt_id, span_id)
search_compressed_spans(query, limit=5)
list_compression_receipts()
```

## New: live proxy installer

Entroly exports a live HTTP proxy installer:

```python
from entroly import install_live_compression_proxy
```

It is intentionally not executed during package import. This keeps wheel builds, Docker quality gates, and downstream library imports side-effect free.

Enable explicitly before starting an embedded proxy path:

```bash
export ENTROLY_COMPRESSION_PROXY_MODE=elc
export ENTROLY_ELC_BUDGET_TOKENS=1200
export ENTROLY_COMPRESSION_STORE=.entroly/compression-store.json
```

## Stability fixes

- Removed import-time live proxy mutation from `entroly.__init__`.
- Added query hygiene so broad words like `build`, `fail`, `log`, and `test` do not accidentally lock thousands of generic lines during compression.
- Kept package imports side-effect free for wheel and Docker CI.
- Preserved exact passthrough behavior when compression mode is off or preserve policies are used.

## Positioning

Entroly v1.0.33 is not just a compression proxy. It is an auditable, recoverable evidence-control plane for compressed LLM context.

The key product wedge:

```text
Headroom retrieves dropped context.
Entroly retrieves dropped context and keeps auditable evidence receipts for what was compressed, why it was compressed, and how to recover it.
```
