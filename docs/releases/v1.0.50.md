# Entroly 1.0.50

Entroly 1.0.50 completes the frontier context-control milestone across
model intelligence, native parity, cryptographic auditability, and
reliable operational delivery.

Highlights:

- provenance-aware model intelligence with conservative unknown-model
  handling, local-model discovery, output-aware context budgeting, and
  effective registry fingerprints;
- Rust-native model resolution and budgeting parity with the Python
  control plane;
- deterministic Model Decision Receipts proving the model metadata and
  budget assumptions used for each model-bearing proxy request;
- durable, idempotent Slack, Discord, and Telegram event delivery with
  SQLite persistence, restart replay, bounded retries, dead-letter
  recovery, destination isolation, redaction, and verifiable receipts;
- synchronized release metadata across Python, Rust, QCCR, WASM, npm,
  MCP, OpenClaw, plugins, generated manifests, lockfiles, Homebrew, and
  release verification tests.

This release preserves Entroly's scope as the local context, memory,
verification, security, and operational control plane for AI agents.
