# Entroly v1.0.36

## Summary

Entroly v1.0.36 completes the live Evidence-Locked Compression integration and adds a guarded native PyO3 fast path.

## Live proxy integration

The HTTP proxy can install ELC compression when:

```bash
ENTROLY_COMPRESSION_PROXY_MODE=elc
```

The hook is installed after `proxy_transform` is initialized and remains a no-op when the feature flag is disabled.

## Guarded PyO3 fast path

`compress_evidence_locked_fast` now:

1. prefers a dedicated `entroly_core.elc_compress` symbol when available,
2. falls back to the existing Rust `py_compress_block` export,
3. accepts native output only when evidence anchors and query-specific lines survive,
4. falls back to the audited Python ELC implementation on ambiguity or error.

The native bridge also checks budget sanity, file-path anchors, diagnostic lines, and empty output.

## Tests

This release adds regression coverage for:

- feature-flagged live proxy hook installation,
- side-effect-free imports when ELC is disabled,
- dedicated native JSON output,
- safe PyO3 compression acceptance,
- native output rejection when evidence is lost.
