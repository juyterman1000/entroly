# Entroly v1.0.37

## Summary

Entroly v1.0.37 adds a cache-aware AI gateway control plane for reducing
provider spend without destroying warm prompt caches or weakening quality
controls.

## Highlights

- deterministic, model-independent stable-prefix construction,
- conversation cache leases with explicit TTL and provider observations,
- multi-turn routing economics for stay-versus-switch decisions,
- risk-specific quality tolerances, hysteresis and escalation overrides,
- capability-safe cross-provider failover planning,
- opt-in outbound redaction with non-reversible receipts,
- durable SQLite usage accounting in integer microdollars,
- OpenAI, Anthropic and Gemini cache-token parsing,
- diminishing-return subagent budget allocation for coding harnesses,
- composed transport-independent `GatewayControlPlane` API.

## Routing economics

The router projects cache-read, cache-write, uncached input and output costs
over a bounded conversation horizon. A model switch occurs only when projected
savings exceed the configured hysteresis and the candidate satisfies the
request's quality and capability constraints.

Provider failure and explicit escalation always override cache stickiness.

## Accounting

`UsageLedger` records provider-reported usage transactionally with idempotent
request IDs. Cache savings remain distinct from compression savings, and every
price carries explicit provenance.

## Compatibility

The existing HTTP proxy continues honoring its no-model-mutation contract.
Applications opt into the new gateway controller explicitly.

All first-party Python, Rust, WASM, npm and Claude plugin manifests are aligned
at version 1.0.37.
