# Entroly 1.0.63

Entroly 1.0.63 adds a trust-bounded efficiency and exact-recovery layer across
the Python SDK, CLI, MCP server, same-origin localhost sidecar, and OpenClaw
context engine.

## Proof-guided context

- Starts with committed selected context, verifies each caller-supplied draft,
  turns unsupported claims into explicit evidence obligations, and recovers
  exact omitted chunks under hard round, token, and chunk limits.
- Adds a hash-sealed, atomic, restart-safe prepare/advance protocol with durable
  idempotency replay. Entroly performs no provider call in this protocol.
- Exposes the same state machine through `entroly proof`, three MCP tools, and
  `/proof/prepare`, `/proof/advance`, and `/proof/inspect` sidecar routes.
- Adds an OpenClaw opt-in that uses typed conversation hooks for local draft
  verification and at most the configured number of host-managed revisions.
  It is disabled by default because a revision may incur another provider call.
- Withholds unsafe, unsupported, or verification-error delivery with an
  actionable message instead of silently passing unverified output.

## Verified efficiency and learning

- Adds signed audit lineage for preparation, output verification, exact
  recovery, fixed-point decisions, and failures.
- Adds a real-transition world-model ledger and uncertainty-bounded synthetic
  proposal ranking. Synthetic transitions remain separate and cannot satisfy
  real promotion or learning gates.
- Separates provider-observed savings, modeled provider savings, local-only
  reductions, and legacy estimates in value accounting.

## Reproduce the contracts

```bash
python benchmarks/verified_efficiency_contract.py
python benchmarks/context_fixed_point_contract.py
python benchmarks/proof_guided_runtime_contract.py
python benchmarks/proof_carrying_dreams_contract.py
```

These are deterministic, no-network conformance checks. They establish control
flow, integrity, recovery, and learning-boundary invariants; they do not claim
model-quality superiority, production savings, or benchmark leadership.

Homebrew remains pinned to the last published sdist until the 1.0.63 PyPI
artifact is available and its SHA-256 has been verified.
