# Entroly v1.0.34

Status: release candidate for the stable Evidence-Locked Compression proxy cut.

## Summary

Entroly v1.0.34 packages the green Evidence-Locked Compression work as a clean new version after the v1.0.33 CI stabilization.

The release keeps package imports side-effect free, preserves the compression proxy APIs, and retains the recoverable compression path introduced in the 1.0.33 line.

## Highlights

- Evidence-Locked Compression for heavy logs, tool output, and JSON payloads.
- Provider-light compression proxy surface for OpenAI-style and Anthropic-style messages.
- Recoverable omitted-span store for compressed context.
- Focused MCP server for retrieving compressed spans.
- Live proxy installer exported without import-time mutation.
- Query hygiene to prevent generic words like `build`, `fail`, `log`, and `test` from locking thousands of low-value lines.
- Version metadata synchronized across `pyproject.toml` and `entroly.__version__`.

## Release positioning

Entroly v1.0.34 is an auditable, recoverable context-compression control plane for agentic AI systems.

The key product wedge remains:

```text
Headroom retrieves dropped context.
Entroly retrieves dropped context and keeps auditable evidence receipts for what was compressed, why it was compressed, and how to recover it.
```
