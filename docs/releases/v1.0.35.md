# Entroly v1.0.35

Status: release candidate for the next compression-control-plane upgrade batch.

## Summary

Entroly v1.0.35 adds the next set of production surfaces around Evidence-Locked Compression:

- dashboard metrics for savings, retrieval, and receipts,
- verification/retry loop for recoverable compressed prompts,
- optional native ELC adapter hook,
- public benchmark table generator,
- direct non-monkey-patch proxy integration helper.

## New: compression dashboard surface

New module:

```python
from entroly import dashboard_from_proxy_receipts
from entroly import dashboard_from_store
```

The dashboard can render JSON or Markdown summaries showing:

- receipts seen,
- compressed blocks,
- original tokens,
- compressed tokens,
- tokens saved,
- savings ratio,
- recoverable spans,
- retrieval receipts.

## New: verification/retry loop

New module:

```python
from entroly import answer_with_retrieval_verification
```

Flow:

```text
compressed prompt -> model answer -> verifier -> retrieve missing span -> retry once
```

The loop is model-agnostic. Callers pass a model callable and optional WITNESS/EICV-style verifier.

## New: optional native ELC adapter

New module:

```python
from entroly import compress_evidence_locked_fast
from entroly import native_elc_available
```

The adapter uses `entroly_core.elc_compress` if the Rust/native symbol is exported. Otherwise it falls back to audited Python ELC.

This keeps the fast-path API stable without falsely claiming native acceleration is active before the Rust export lands.

## New: public benchmark table generator

New script:

```bash
python benchmarks/compression_public_table.py --write docs/benchmarks/compression-proxy-table.md
```

The generator renders a Markdown table from deterministic scoreboard scenarios. It gates both token savings and evidence preservation.

## New: direct proxy integration helper

New module:

```python
from entroly import apply_elc_to_proxy_body
```

This is the direct non-monkey-patch integration point for `proxy.py`. It applies ELC to the request body and returns transformed body, receipt, and headers.

## Honest note

The Rust native adapter is ready, but actual Rust acceleration requires exporting a compatible `elc_compress` symbol from `entroly-core`.

The direct proxy helper is ready, but the literal `proxy.py` request-handler patch should be applied with a normal git patch rather than rewriting the whole large file through the connector.
