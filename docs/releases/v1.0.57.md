# Entroly 1.0.57

Entroly 1.0.57 makes the OpenClaw context engine provider-independent at the
normalized-message boundary. OpenAI, Anthropic, Gemini, Nemotron, OpenRouter,
Ollama, and private routes use the same host-authoritative budget path while
OpenClaw retains routing, authentication, compaction, and overflow recovery.

The bridge now compresses eligible unsigned text inside normalized assistant
and tool-result blocks while preserving signed text, thinking, images, tool
calls, identifiers, provider metadata, and recent/system/developer messages
exactly. Invalid host envelopes are rejected, unsafe or malformed bridge
responses fail open to the original context, and over-budget minimum context is
returned with explicit OpenClaw recovery authority.

OpenClaw receipts use the v2 append-only proposal protocol. Each proposal has a
unique nonce and immutable digest, survives bridge restarts, is committed
idempotently only after JavaScript validation reveals a precommitted 256-bit
challenge. The secret is never persisted: Entroly records a private-key HMAC
signature and HMAC-keyed content digests instead. Receipts store query lengths
and match counts without plaintext queries, query-derived receipt identities,
offline-guessable content hashes, or per-term digests. Default stores
reject workspace symlink escapes. Private Unix modes, durable atomic renames,
and fixed-size non-destructive file/byte quotas prevent silent exposure,
partial audit commits, or unbounded growth.

Marketplace publication is coordinated behind the matching Python release. CI
installs the minimum supported OpenClaw host, exercises the real JSONL bridge,
and publication waits until clean PyPI and npm installs prove the required v2
two-phase capability before ClawHub is updated. Manual release versions are
strictly validated once and must match every committed package surface.
